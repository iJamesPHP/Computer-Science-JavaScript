Analysis

So, I need to develop a program where in which the user can first tell the program what device and model they are having issues with so that the program can load a different set of suitable question-answer pairs based on their device.
This program needs to incorporate a mechanism that transforms the user's input about their device into a concatendated filename. For example: if the user types 'iphone' for the device, and typed '5s' for the model, the program will load iphone5s.txt
I'll also need to write logic that will create a file with their device information and issue as well as assign them a case number for a support agent to access at a later date.

Core Logic
In this section I'll delve deeper into my core logic and how my code works proceduraly.
Now I get into the core logic of the program, it works by first taking the user's input for their device name and the device model, the program then concatenates these two bits of data and appends '.txt' onto the end. If this file name doesnt exists, it's an unsupported device and the program exits. If this file name does exist, it's a supported device and the file is read. The user then types their issue. Then each line of the file is looped over and if it doesnt contain the user's issue then their device info and issue are stored in a .txt file with a case number for an agent, if it does contain the user's issue then the relevant solution in the file is printed and the program exits.

I chose to use these logical steps in my program because it let's me cover the most important requirement, multi-device support via file accessing. By concatenating the user's inputted data to load seperate issue-solution pair files, I can tailor the solutions to the user's issues based on their device.

I have files for these devices:
  • iphone5s
  • iphone5c

I could add more files to add support for more devices easily, this is good because it makes my program easily expandable.
